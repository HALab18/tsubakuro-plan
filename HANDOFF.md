## 1. このリポジトリは何か

仙台発・3名の燕岳（1泊2日／燕山荘泊）登山プランの共有ページ。

| 項目 | 値 |
|------|-----|
| GitHub | https://github.com/HALab18/tsubakuro-plan |
| 公開URL | https://halab18.github.io/tsubakuro-plan/ |
| 公開方式 | GitHub Pages（master ブランチ / ルート） |
| ブランチ | master |

## 2. ファイル構成

| ファイル | 役割 |
|----------|------|
| `index.html` | **プランページ本体。単一HTML完結（CSS内包・外部依存なし）。実質ここだけを編集する** |
| `DESIGN_SPEC.md` | 「わくわくパステル」テーマのデザイン仕様書。色・コンポーネントCSSの引き継ぎ用 |
| `HANDOFF.md` | このファイル |

※ 過去の tozan-plan スキルの手順には「`<repo名>.html` を作って `index.html` にコピー」とあるが、**このリポジトリは `index.html` 単独運用**。二重管理はしていないので、`index.html` をそのまま編集すればよい。

## 3. 自宅で編集を再開する手順

```bash
git clone https://github.com/HALab18/tsubakuro-plan.git
cd tsubakuro-plan
# index.html をエディタで編集。ブラウザで index.html を直接開けばプレビューできる
git add index.html
git commit -m "説明"
git push
```

push から数分で https://halab18.github.io/tsubakuro-plan/ に反映される。

## 4. index.html の構造（上から順）

1. `<style>` — CSS変数 `:root` ＋ 全コンポーネントのCSS（`DESIGN_SPEC.md` と対応）
2. `.hero` — タイトル・日付ピル
3. `.yamap-banner` — YAMAP計画へのリンク（緑のバナー）
4. タイムスケジュール — `.card` × 2（1日目 `card-header day1` / 2日目 `card-header day2`）
   - 各行は `.sch-row` ＞ `.sch-time` ＋ `.sch-body`（`.sch-main` ＋ `.sch-note`）
   - 行の色分け: `hl-bus`（青＝バス） / `hl-hike`（ピンク＝登山） / `hl-onsen`（アンバー＝温泉）
5. バス時刻表 — `.bus-table`。**今回乗車便の行に `class="star"` と `★` を付ける**（上り・下りで各1本）
6. 施設情報 — 燕山荘 / 大王わさび農場 / しゃくなげの湯 / タイムズ安曇野穂高
7. 帰り道の立ち寄りスポット — 道の駅あらい / 姨捨SA / 米山SA / 阿賀野川SA
8. 費用 — `.metric-grid`（サマリー2枚）＋ ①割り勘 / ②HAL単独 / ③全員共通
9. 持ち物 → リンク集 → フッター

**編集時の注意**: 色を変えるときは必ず `-text` 系の濃い変数を文字色に使う（薄ピンク `--col-accent` を文字に使わない）。詳細は `DESIGN_SPEC.md` の「コントラストのルール」。
